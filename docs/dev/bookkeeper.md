# 记账员 - 设计思路

## 项目概述

一个提供对话式 journalize 功能的 AI Agent。用户通过自然语言描述交易，AI 自动生成复式记账格式并写入账本。

**设计理念**：
> 让大模型做打字员，让解析器做审核员，让代码保持简单

---

## 核心架构

### 分层设计

```
┌─────────────────────────────────────┐
│           UI 层 (TUI)               │  用户交互
├─────────────────────────────────────┤
│         业务逻辑层                   │  journalize()
├─────────────────────────────────────┤
│    ┌─────────────┐  ┌────────────┐  │
│    │  Prompt     │  │  LLM       │  │  核心能力
│    │  Builder   │  │  Caller    │  │
│    └─────────────┘  └────────────┘  │
├─────────────────────────────────────┤
│    ┌─────────────┐  ┌────────────┐  │
│    │  Extractor │  │  Validator │  │  解析与验证
│    └─────────────┘  └────────────┘  │
├─────────────────────────────────────┤
│           存储层                     │  本地/云端账本
└─────────────────────────────────────┘
```

### 模块职责

| 模块 | 职责 | 边界 |
|------|------|------|
| **Prompt Builder** | 构建系统提示词 + 用户提示词 | 输入：账户列表 + 用户描述；输出：prompt |
| **LLM Caller** | 调用 LLM API | 输入：prompt；输出：文本响应 |
| **Extractor** | 从 LLM 输出提取目标格式 | 输入：文本；输出：结构化内容 |
| **Validator** | 验证语法/语义正确性 | 输入：结构化内容；输出：是否有效 |
| **Storage** | 账本读写 | 输入：路径；输出：内容 |

## 核心接口

### journalize()

```
输入：
  - user_input: string  (用户自然语言描述)
  - ledger_path: optional<path>  (账本路径)

输出：
  - success: boolean
  - message: string (结果或错误信息)

前置条件：
  1. 账本文件存在或可创建
  2. LLM 服务可用

后置条件：
  1. 成功时：账本文件追加新交易
  2. 失败时：账本文件不变
```

## 验证策略

### 双重验证

1. **语法验证**：使用原生解析器检查格式正确性
2. **语义验证**（可选）：
   - 借贷平衡
   - 账户存在
   - 日期格式

**原则**：验证失败时拒绝写入，保证账本一致性。
